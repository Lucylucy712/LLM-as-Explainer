# LLM as Explainer: Feature-Ranking Fidelity Experiments

This repository contains the notebooks and reproducibility artifacts for evaluating whether large language models preserve model-provided feature-importance rankings in instance-level explanations.

The project studies two base predictive models, logistic regression and XGBoost, and compares LLM-generated rankings against model-derived reference rankings from coefficient/SHAP-style contribution tables.

## Research Questions

- **RQ1: Ranking preservation.** Given an ordered model-provided feature-importance list, does the LLM reproduce the supplied ranking?
- **RQ2: Feature identification.** Given instance information and model attribution values, can the LLM identify the most important features? RQ2 includes Top-4 and full-list conditions, under zero-shot and few-shot prompt settings.

## Environment

The notebooks were prepared with Python 3.10. A conda environment file is provided:

```bash
conda env create -f environment.yml
conda activate llm-explainer
python -m ipykernel install --user --name llm-explainer --display-name "llm-explainer"
```

Then select the `llm-explainer` kernel in Jupyter or VS Code.

## API Keys

LLM calls use environment variables loaded from a local `.env` file. Do **not** commit your real `.env` file. Use `.env.example` as a template:

```bash
cp .env.example .env
```

Fill in whichever providers you plan to run:

```text
OPENAI_API_KEY=...
ANTHROPIC_API_KEY=...
GEMINI_API_KEY=...
```

The included `.gitignore` excludes `.env` and local virtual environments.

## Reproduction Workflow

A typical workflow is:

1. Run `Code/Step1_LoadData.ipynb` to prepare the cleaned dataset, if starting from raw data.
2. Run `Code/Step2_BaselineModel.ipynb` to train the base models.
3. Run `Code/Step3_SHAP.ipynb` to compute and aggregate feature contributions.
4. Run the redesigned RQ1 notebooks for ranking-preservation experiments.
5. Run the redesigned RQ2 notebooks for zero-shot and few-shot experiments.
6. Run the summary notebooks to produce paper-ready tables and bootstrap confidence intervals.

The `Result/` folder includes selected intermediate and summary artifacts so the analysis notebooks can be inspected without rerunning every LLM call.

## Metrics

- **Overlap@K:** fraction of reference Top-K features recovered by the LLM Top-K list.
- **Kendall's tau@K:** rank correlation computed over the intersection of the reference Top-K and LLM Top-K feature sets. If fewer than two overlapping features are available, tau is undefined.
- **Schema-level fidelity:** checks whether the response is parseable JSON, uses only supplied feature names, has no duplicate features, and returns the required number of ranked positions.
- **Bootstrap confidence intervals:** 95% CIs are computed by bootstrapping over instances.

## Notes for Open Sourcing

- Raw API keys and local environments are excluded.
- Raw LLM batch outputs were removed from this GitHub-ready copy to keep the repository compact; summary metrics and paper-ready tables are retained.
- Model pickle files are excluded because they are regenerated by Step 2 and can be version-sensitive.
